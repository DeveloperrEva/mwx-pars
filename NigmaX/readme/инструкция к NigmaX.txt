Как указывать аргументы:
Флаги указываются ДО агрумента процедуры. Пример: --details test
Команды указываются ПОСЛЕ аргумента процедуры. Пример: update [all]
Процедуры указываются в удобном месте. Остальные аргументы вводятся относительно них.
Данные, которые может передавать флаг/комманда указываются ПОСЛЕ этого же флага/команды. Пример: --dbname nigmax.sqlite
В справочнике изменяемые данные для примера показаны в скобках {}. Вводить их не нужно. Пример: --MUid {MCD id}
Обычные и сокращенные аргументы разделены знаком |. Одновременно использовать только один из них.

Глобальные флаги:
{path_to_parser.json}   - Путь к json-парсеру
--dbname *.sqlite       - Задать имя базы данных (по умолчанию: nigmax.sqlite)
--skipauth    | --sa    - Пропустить авторизацию на сайте
--skipmlctest | --smlct - Пропустить тестовую [manga_list_complete] процедуру
--skipmctest  | --smct  - Пропустить тестовую [manga_complete] процедуру
--save        | --s     - Сохранять загруженные данные в html файлы

Процедуры:
test - Проводит тестирование парсеров
 Команды:
  [manga_list_complete]  - Запустить тест [manga_list_complete] процедуры
  [manga_complete]       - Запустить тест [manga_complete] процедуры
  [chapter_complete]     - Запустить тест [chapter_complete] процедуры
  [page_complete]        - Запустить тест [page_complete] процедуры
   Флаги:
    --all {путь_к_папке} - Запустить тесты всех json парсеров в указанном каталоге
    --details | --d      - Включить вывод логов в консоль

update - Обновляет список манги и основные данные манги в базе данных
 Команды:
  [all]                  - Обновить список и данные манги в базе данных
  [manga_list_complete]  - Обновить список манги в базе данных
  [manga_complete]       - Обновить данные каждой манги в базе данных
   Флаги:
    --all                   - Выполнить ВСЕ процедуры по обновлению списка и данных манги
    --dontremoveold | --dro - Не удалять старую мангу при выполнении процедуры [manga_list_complete]
    --removebroken  | --rb  - Удалять сломанную мангу при выполнении процедуры [manga_complete]
    --delaymlc | --dmlc {x} - Задержка процедуры парсинга [manga_list_complete] на {x} сек
    --delaymc  | --dmc {x}  - Задержка процедуры парсинга [manga_complete] на {x} сек
    --qbd {x}               - Кол-во необходимых запросов к серверу перед задержкой процедуры [manga_complete]



export     - Экспортировать список манги в ProtoBuf (*.p) файл

Дополнительные процедуры:
status     - Количество манги для json файла
statusall  - Количество всей манги в базе данных
clearall   - Очистить всю базу данных
lastupdate - Получение времени последнего изменения базы данных
logout     - Разлогин с указанного сайта
